# MEDIA RELIC v0.1

**MEDIA RELIC v0.1 by Alex Merqury**  
**Delirium Interactive**

Minimalist Unicode media relic for Windows.

It uses:

- `mpv.exe` for playback.
- `ffmpeg.exe` for silence detection, segment export, and Unicode preview sampling.
- `ffprobe.exe` for duration probing.

No NuGet packages. No mega-script. The code is split into small modules, because civilization occasionally remembers restraint.

## Folder layout

```text
MediaRelic_v0_1/
  MediaRelic.csproj
  src/
    App/        command/config/app-core/logging
    Domain/     tiny data objects
    Infra/      mpv, ffmpeg, process, tools
    Playlist/   folder scanning and filtering
    Ui/         WinForms shell and Unicode canvas
  tools/
    put mpv.exe here
    put ffmpeg.exe here
    put ffprobe.exe here
```

The tools may also be available globally in PATH.

## Build

Install .NET 8 SDK for Windows.

```powershell
dotnet run -c Release
```

Or build:

```powershell
dotnet publish -c Release -r win-x64 --self-contained false
```

## Living UI

The text UI has a lightweight idle animation:

```text
- tiny color pulse on borders/status/header
- occasional decorative glyph morph on frame/progress lines
- no per-pixel particle soup
- no extra worker thread
```

It rides on the existing 250 ms UI timer. Cheap, visible, and less likely to turn the player into a tiny space heater.

## Architecture pass

The app is now split so the UI is intentionally dumb:

```text
Input -> MainForm -> RelicApp -> mpv / ffmpeg / preview / playlist
State -> RelicCanvas -> Unicode UI
```

Important pieces:

```text
App/RelicApp.cs          owns app flow, playback actions, playlist actions, scan/export actions
App/RelicConfig.cs       loads optional runtime settings
App/RelicLogger.cs       writes MediaRelic.log without killing playback if logging fails
App/RelicCommand.cs      explicit command vocabulary for future input remapping
Domain/RelicMode.cs      explicit app mode instead of magic bool soup
Playlist/PlaylistScanner.cs filters folders and rejects files shorter than the configured limit
Ui/MainForm.cs           thin shell: dialogs, key mapping, drag/drop, window resize/move
```

`MainForm` should not grow tentacles again. If a new feature needs mpv, ffmpeg, preview, playlist, or export logic, it belongs outside `Ui/`. Humanity has suffered enough from god-forms.

## Runtime config

Defaults are compiled in, but you can copy `mediarelic.config.example.json` to `mediarelic.config.json` and tune:

```json
{
  "minPlaylistDurationSeconds": 15,
  "previewMaxWidth": 96,
  "previewMaxHeight": 96,
  "defaultVolume": 100,
  "startTopMost": false,
  "uiScale": 1.0
}
```

The config loader checks the current folder and the app folder.

## Silence slicing

Default scan:

```text
noise: -38 dB
duration: 0.35 s
```

The app builds sound ranges between silence blocks and exports them as WAV chunks into:

```text
MediaRelic_Cuts/
```

## Unicode preview

If the file contains embedded cover art, MEDIA RELIC uses that first. For MP3 this usually means the ID3 APIC cover image stored inside the file itself.

If there is no embedded cover, MEDIA RELIC checks for a cover image next to the track. You can create this manually from inside the player with `C`. Supported names: `same-name-as-track.jpg/png`, `cover`, `folder`, `front`, `album`, `artwork`, `poster` with `.jpg`, `.jpeg`, `.png`, `.webp`, `.bmp`.

If there is no cover art but the file has video, the preview is generated by sampling the current video frame with FFmpeg.

The image is converted into colored Unicode glyphs:

```text
brightness -> ASCII/Unicode density ramp
edges      -> │ ─ ╱ ╲
color      -> glyph foreground color
```

For audio-only files without cover art, MEDIA RELIC shows a static `NO COVER ART` card. No fake spectrum, no decorative waveform cosplay.

## 96 rune cover mode

Cover preview now uses a maximum grid of:

```text
96 × 96 glyph cells
```

The glyph grid follows the real image aspect ratio:

```text
glyph_columns / glyph_rows ~= image_width / image_height
```

Examples:

```text
184 × 184 cover  -> square Unicode frame, up to 96 × 96 cells
1920 × 1080 art  -> 16:9 Unicode frame
800 × 1200 art   -> portrait Unicode frame
```

`256 × 256` was impressive in the way a dump truck in a living room is impressive. This version keeps the stylized Unicode look without making the UI obese.

## Folder playlist

Press `P` or drag a folder into the window. MEDIA RELIC scans only the top folder level and accepts supported media files with duration `>= 15 seconds` by default.

Tiny trash files under the configured duration are skipped.

Use `N` and `B` for next/previous. When a track ends, the next item starts automatically unless loop is enabled.

## Closing behavior

`Esc`, `Q`, and the custom `×` button stop playback and close the app.

## Applying a cover manually

Press `C` while a track is loaded.

MEDIA RELIC lets you pick:

```text
.jpg
.jpeg
.png
.webp
.bmp
```

The chosen image is copied next to the current track as:

```text
Track Name.png
Track Name.jpg
...
```

This does not rewrite the MP3. It is a safe sidecar cover for MEDIA RELIC preview. Rewriting audio files just to paint a tiny JPEG into metadata is how software grows tumors.

Embedded MP3 covers still have priority. If a file has no embedded cover, the sidecar image is used.
