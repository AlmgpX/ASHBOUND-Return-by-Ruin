# MEDIA RELIC v0.1

**MEDIA RELIC v0.1 by Alex Merqury**  
**Delirium Interactive**

Minimalist always-on-top Unicode media relic for Windows.

It uses:

- `mpv.exe` for playback.
- `ffmpeg.exe` for silence detection, segment export, and Unicode preview sampling.
- `ffprobe.exe` for duration probing.

No NuGet packages. No mega-script. The code is split into small modules, because civilization occasionally remembers restraint.

```

dotnet publish -c Release -r win-x64 --self-contained false
```

## Silence slicing

Default scan:

```text
noise: -38 dB
duration: 0.35 s
```

The app builds sound ranges between silence blocks and exports them as WAV chunks into:

```text
MediaRelic_Cuts/
```

## Unicode preview

If the file contains embedded cover art, MEDIA RELIC uses that first. For MP3 this usually means the ID3 APIC cover image stored inside the file itself.

If there is no embedded cover, MEDIA RELIC checks for a cover image next to the track. You can create this manually from inside the player with `C`. Supported names: `same-name-as-track.jpg/png`, `cover`, `folder`, `front`, `album`, `artwork`, `poster` with `.jpg`, `.jpeg`, `.png`, `.webp`, `.bmp`.

If there is no cover art but the file has video, the preview is generated by sampling the current video frame with FFmpeg.

The image is converted into colored Unicode glyphs:

```text
brightness ->  ·░▒▓█
edges      ->  │ ─ ╱ ╲
color      ->  glyph foreground color
```

For audio-only files without cover art, MEDIA RELIC shows a static `NO COVER ART` card. No fake spectrum, no decorative waveform cosplay.

## Folder playlist

Press `P` or drag a folder into the window. MEDIA RELIC scans only the top folder level and accepts supported media files with duration `>= 15 seconds`.

Tiny trash files under 15 seconds are skipped.

Use `N` and `B` for next/previous. When a track ends, the next item starts automatically unless loop is enabled.

## Closing behavior

`Esc`, `Q`, and the custom `×` button stop playback and close the app.
## Applying a cover manually

Press `C` while a track is loaded.

MEDIA RELIC lets you pick:

```text
.jpg
.jpeg
.png
.webp
.bmp
```

The chosen image is copied next to the current track as:

```text
Track Name.png
Track Name.jpg
...
```

This does not rewrite the MP3. It is a safe sidecar cover for MEDIA RELIC preview. Rewriting audio files just to paint a tiny JPEG into metadata is how software grows tumors.

Embedded MP3 covers still have priority. If a file has no embedded cover, the sidecar image is used.


## Cover preview density / proportions

This patch makes the cover preview denser and more faithful:

- preview grid increased to `96 x 36`
- smaller, tighter preview glyphs
- cover aspect ratio is preserved inside the preview box
- the image is not forced to a naive pixel-for-symbol mapping
- better scaling (`lanczos` / bicubic) is used before Unicode conversion

This is specifically meant to help logos and cover text stay more readable.
## Aspect-locked dense cover mode

Cover preview is rendered with dense Unicode half-blocks:

```text
▀ = upper half uses one sampled color, lower half uses background color
```

The preview grid is no longer fixed at `112 × 44`.

MEDIA RELIC now computes the Unicode frame size from the actual image pixel ratio:

```text
glyph_columns / glyph_rows ~= image_width / image_height
```

Examples:

```text
184 × 184 cover  -> square Unicode frame
1920 × 1080 art  -> 16:9 Unicode frame
800 × 1200 art   -> portrait Unicode frame
```

The current maximum preview box is:

```text
96 × 96 glyph cells
```

So a square cover becomes roughly `96 × 96`, not a flattened rectangle. Finally, a square is allowed to remain a square. Revolutionary technology, apparently.
